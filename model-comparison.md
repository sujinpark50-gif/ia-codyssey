# 모델 비교·선정 보고서

## 1. 비교 목적

요구사항 분석 업무에 가장 적합한 LLM을 선정하기 위해 GPT-5.5, Claude Sonnet 4.6, Gemini 3.5 Flash를 비교하였다.

비교는 동일한 페르소나, 동일한 출력 형식, 동일한 테스트 시나리오를 적용하여 수행하였다.

## 4. 평가 기준

### 1) 정확성

사용자가 제공한 정보를 정확하게 해석하고 없는 내용을 임의로 생성하지 않는가

### 2) 형식 준수

지정된 출력 형식을 일관되게 준수하는가

### 3) 정보 부족 판단 능력

정보가 부족할 때 추가 질문을 제시하는가

### 4) MVP 설계 능력

예산과 일정 제약을 고려하여 현실적인 MVP를 제안하는가

### 5) 실무 활용성

실제 개발 및 기획 문서로 활용 가능한 수준인가

### 6) 한국어 자연스러움

문서가 자연스럽고 읽기 쉬운가?

## 근거

### GPT-5.5

* 요구사항을 체계적으로 구조화하였다.
* 정보 부족 상황에서도 분석 가능한 영역과 불가능한 영역을  명확히 구분하였다.
* MVP 범위를 현실적으로 정의하였다.
* 일부 기능을 자연스럽게 확장하는 경향이 있었으나 전체적으로 정확도가 높았다.

### Claude Sonnet 4.6

* 정보 부족 상황에서 임의의 기능을 생성하지 않고 추가 질문을 우선하였다.
* 사용자 요구사항과 AI 제안 사항을 명확히 구분하였다.
* CafeStamp 시나리오에서 쿠폰 유효기간, 인증 방식 등 누락 요구사항을 식별하였다.
* 세 시나리오 모두에서 요구사항 분석가 역할을 가장 일관되게 수행하였다.

### Gemini 3.5 Flash

* 문서 구조와 가독성은 우수하였다.
* 다만 정보 부족 상황에서 추가 질문보다 일반적인 해결책을 먼저 제안하는 경향이 관찰되었다.
* 요구사항 분석보다는 컨설팅 성향이 상대적으로 강하게 나타났다.

## 최종 선정 모델

Claude Sonnet 4.6

### 선정 이유

1. 정보 부족 상황에서 추가 질문을 통해 요구사항을 수집하려는 태도가 가장 우수하였다.
2. 사용자 요구사항과 AI 제안을 명확하게 구분하여 문서의 신뢰성이 높았다.
3. 세 가지 테스트 시나리오에서 가장 일관되게 요구사항 분석가 역할을 수행하였다.
4. 요구사항 분석 업무 특성상 정확성과 형식 준수, 정보 부족 판단 능력이 중요하다고 판단하여 최종 선정하였다.
